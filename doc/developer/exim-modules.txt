EXportIMport - набор модулей для организации частичного обмена данными между несколькими системами Veda

ноды - это системы veda настроенные для обмена друг с другом
обмен происходит по схеме master/slave, где master производит обмен с нодами slave



назначение модулей

veda-extractor :
	 читает текущую очередь queue/individuals-flow и выгружает прошедшие фильтр обьекты в очередь out/extractor  

veda-exim-inquire : 
	Master, считывает описание нод, производит попытки отправки накопленных данных из очереди out/extractor. Так же делает запросы к нодам на прием данных.
	Принятые данные сохраняются в базе данных с меткой ID slave ноды         

veda-exim-respond :
	Слушает порт приема данных и реагирует на прием данных либо запрос на отправку данных из slave.
	При приеме сохраняет принятые обьекты в базу данных с меткой ID master ноды.
	При запросе измененных данных производит их отправку в ответном сообщении.

прием/отправка производится по протоколу nanomsg в режиме reqrep

конфигурация master ноды
	veda-extractor + veda-exim-inquire

конфигурация slave ноды
	veda-extractor + veda-exim-respond

одна master нода может взаимодействавать с множеством slave нод
