описание обработки очереди

модуль обрабатывающий очередь должен реализовать следующие функции: before_batch(), prepare(), after_batch()

алгоритм обработки:

Модуль находится в бесконечном цикле и ожидает сигнала из вне о том что файл очереди изменился. 
При получении сигнала или если сигнал не поступает в течении заданного времени наступает цикл чтения данных из очереди.
Перед началом чтения происходит вызов функции before_batch(), далее будет выполнятся вызов функции prepare() для каждого считанного элемента очереди 
Чтение будет выполнятся до тех пор пока читатель не дойдет до конца файла очереди.
После окончания чтения происходит вызов функции after_batch()
Далее модуль снова переходит в режим ожидания новой порции данных из очереди.
